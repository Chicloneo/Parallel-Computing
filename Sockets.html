Introduction

We explain in this document how the connection client-server works. This is a very simple introduction with Pythonusing the socketlibrary. We will have two files: socket_cliente.py and socket_servidor.py, in order to distinguish between the client and the server, respectively.

Basic commands

The socket library must be imported.

from socket import socket

socket_cliente.py

sckt = socket()                      # creates a socket
srvr_ip = "localhost"                # Server IP address. We want to connect to "localhost"
srvr_puerto = 54323                  # Server is listening through this port
sckt.connect((srvr_ip, srvr_puerto)) # Connecting to the server.

The server IP is "localhost" because this is a simulation on a single computer.

You may change srvr_puerto if you get some connection errors. This should be a number between 1024 and 99999.

Now that the client and server are connected, the information sending and recieving should occur.

To end the connection run this line

sckt.close()

The condition is simple. While the user does not want to quit, the server remains avaliable for the user.

The user enters the command UPPER or LOWER, depending on what he wants to do. Then he enters the text. The client send the command and text information together. It is up to the server to split the information. Then we will recieve the text converted to upper or lower.

Server

msj_rec = cl_socket.recv(1024)       # The server recieves the message, 1024 bytes max.
while msj_rec:

    peticion = msj_rec.decode()      # We decode the message

    lista_datos = peticion.split(':')
    comando = lista_datos[0]
    texto = lista_datos[1]

    if comando == 'UPPER':
        respuesta = texto.upper()
    elif comando == 'LOWER':

    else:
        respuesta = ('ERROR: petición no reconocida.')

    cl_socket.send(respuesta.encode())
    msj_rec = cl_socket.recv(1024)

How does the condicion while msj_rec: work? The loop is actually asking “do you have any information?”. So if the message is empty, None, then the condition is False, and the loop ends.

We split the information into UPPER/LOWER and the text. Then the server does the job depending on th ecommand. Using elif is essential. Do not use if. Imagine you have two ifs and the command is UPPER. The first condition holds. The second does not, so it will enter the else condition and an error will show up.

Multiple Users

We may now consider multiple users entering the server and sending commands. The server has to manage this situation and it cannot give information to the wrong user. We solve this by using Threads.

The server scrpit will look like this:

while True:
    cl_socket, _ = srvr_socket.accept()
    cl_thread = AtiendeCliente(cl_socket)
    cl_thread.start()

The server first listens and accepts the clients (N users simultaneously). Then it creates the threads. Finally it assists the users.

while True is implemented because servers are meant to be avaliable at any time. For this reason, we can run our socket_cliente.py as much times as we want. The servers will be always waiting. The only way to end is to kill the terminal.

We create the following class to assist the client in the server script:

class AtiendeCliente(Thread):
    def __init__(self, cl_socket):
        Thread.__init__(self)       # Create the thread
        self.cl_socket = cl_socket

    def run(self):
        msj_rec = self.cl_socket.recv(1024) 
        while msj_rec:

            peticion = msj_rec.decode()

            lista_datos = peticion.split(':')
            comando = lista_datos[0]
            texto = lista_datos[1]

            if comando == 'UPPER':
                respuesta = texto.upper()
            elif comando == 'LOWER': #Es importante un elif y no un if, porque si el comando es UPPER, volvería a preguntar si es LOWER. Como no lo es, entra en el else y devuelve error
                respuesta = texto.lower()
            else:
                respuesta = ('ERROR: petición no reconocida.')

            self.cl_socket.send(respuesta.encode())
            msj_rec = self.cl_socket.recv(1024)

        self.cl_socket.close()

The run structure is the same as we saw before. Remark AtiendeCliente is a Thread subclass. AtiendeCliente has to know which client it has to help, so it recieves the client socket as the input.

Turning off the server

We consider the situation where a user requests to turn the server off (and it will do so).

We modify the AtiendeCliente class initialization in the server script.

class AtiendeCliente(Thread):
    def __init__(self, cl_socket, almacen: Almacen, parar_event: threading.Event):
        Thread.__init__(self)
        self.cl_socket = cl_socket
        self.almacen = almacen
        self.parar_event = parar_event

We modify the run method.

elif comando == 'SHUTDOWN':
    respuesta = 'Servidor apagándose...'
    self.parar_event.set()

The server manages the situation now like this. It has set a 5 seconds limit in order to recieve any command. In other case, it will shut down.

while not hay_que_parar.is_set():
    try:
        cl_socket, _ = srvr_socket.accept()
        cl_socket.settimeout(5)
        cl_thread = AtiendeCliente(cl_socket, almacen_compartido, hay_que_parar)
        cl_thread.start()
    except TimeoutError:
        print("No se han recibido conexiones en los últimos 5 segundos. Cerrando el servidor.")

You can think of .set() as saying “I want to stop the event”. So in the next loop iteration the condition will not hold.
